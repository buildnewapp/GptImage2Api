## SEO & Branding Templates

### Title Patterns

- **Product-focused:** `{ProductName}`
- **With category:** `{ProductName} - {Category}`

## Localization Best Practices

**Core Principle: Write for natives, don't translate for foreigners.**

Each language version should feel like it was originally written in that language. A Chinese user should read the Chinese version and think "this company understands Chinese users," not "this is translated from English."

### English (en) Guidelines

**Tone & Style:**
- Direct, benefit-focused, action-oriented
- Use active voice: "Build your app" not "Your app can be built"
- Front-load benefits in sentences
- Use power words: "Launch", "Transform", "Accelerate", "Unlock"

**SEO Best Practices:**
- Primary keyword in first 60 characters of title
- Natural keyword density (1-2%)
- Use semantic variations, not repetition
- Meta description: compelling hook + key benefit + CTA hint

**Example Patterns:**
```
Title: "{ProductName} - {Primary Keyword} for {Audience}"
Tagline: "{Action Verb} {Benefit} {Timeframe/Ease}"
Description: "{ProductName} helps {audience} {achieve result}. {Key feature}. {Differentiator}."
```

### Chinese (zh-CN) Guidelines

**语气与风格:**
- 专业但亲切，像一位可信赖的朋友在推荐
- 避免生硬的翻译腔和过于书面的表达
- 使用中国互联网用户熟悉的表达方式

**本土化表达对照:**

| 英文原文 | ❌ 直译 | ✅ 本土化 |
|----------|---------|----------|
| Get Started | 获取开始 | 立即开始 / 免费试用 |
| Launch faster | 更快地启动 | 快速上线 |
| Build in days, not months | 在几天内构建而不是几个月 | 几天交付，告别漫长等待 |
| Transform your business | 转变你的业务 | 让业务焕然一新 |
| Production-ready | 生产就绪的 | 开箱即用 |
| One-click deploy | 一键部署 | 一键部署 ✓ (这个OK) |
| Trusted by developers | 被开发者信任 | 深受开发者信赖 |
| All-in-one solution | 一体化解决方案 | 一站式解决方案 |

**SEO 关键词策略:**
- 使用中国用户实际搜索的词汇
- 考虑百度搜索习惯（关键词靠前）
- 标题格式: `{产品名} - {核心关键词}｜{差异化卖点}`
- 避免过度堆砌，保持自然阅读体验

**技术术语处理:**
| English | Chinese | Notes |
|---------|---------|-------|
| API | API | 保持英文 |
| SaaS | SaaS | 保持英文，可补充"软件即服务" |
| Dashboard | 仪表盘 / 控制台 | 根据上下文选择 |
| Authentication | 身份认证 / 登录认证 | |
| Subscription | 订阅 | |
| One-time payment | 一次性付款 / 买断制 | |
| Boilerplate | 模板 / 脚手架 | |
| Deploy | 部署 | |
| Checkout | 结账 / 支付 | |

**常见错误:**
- ❌ 使用"您"过多显得生硬 → 适当使用"你"更亲切
- ❌ 句子过长，从句嵌套 → 短句为主，清晰直接
- ❌ 英文标点混用 → 使用中文标点（，。！？）

### Japanese (ja) Guidelines

**語調とスタイル:**
- です/ます調（敬語）を基本とする
- 過度にカジュアルにならず、信頼感を維持
- 日本のビジネス文化に配慮した表現

**ローカライズ表現対照:**

| 英語原文 | ❌ 直訳 | ✅ ローカライズ |
|----------|---------|----------------|
| Get Started | 開始する | 今すぐ始める / 無料で試す |
| Launch faster | より速くローンチする | 素早くリリース |
| Build in days | 数日で構築 | 数日で完成 |
| Transform your business | ビジネスを変革する | ビジネスを次のステージへ |
| Production-ready | 本番環境対応 | すぐに使える |
| Trusted by developers | 開発者に信頼されている | 多くの開発者に選ばれています |

**SEO キーワード戦略:**
- 日本のユーザーが実際に検索する言葉を使用
- タイトル: `{製品名}｜{メインキーワード} - {ベネフィット}`
- 検索意図に合わせたキーワード選定
- 自然な文章の中にキーワードを配置

**技術用語の扱い:**
| English | Japanese | Notes |
|---------|----------|-------|
| API | API | 英語のまま |
| SaaS | SaaS | 英語のまま |
| Dashboard | ダッシュボード | カタカナ |
| Authentication | 認証 | |
| Subscription | サブスクリプション | カタカナが一般的 |
| Template | テンプレート | |
| Deploy | デプロイ | 開発者向けはカタカナ |

**日本市場特有の配慮:**
- 控えめな表現を好む傾向（"10倍速い"より"大幅に効率化"）
- 信頼性・実績を重視する文化
- 具体的な数字やデータが好まれる
- 「〜できます」「〜いただけます」など可能表現を活用

**よくある間違い:**
- ❌ 過度にカジュアルな表現 → ビジネスシーンでは敬語を維持
- ❌ 直訳調の不自然な語順 → 日本語として自然な語順に
- ❌ 漢字が多すぎて読みにくい → 適度にひらがなを混ぜる

### SEO Content Quality Checklist

Before finalizing content for any language:

- [ ] Read aloud: Does it sound natural to a native speaker?
- [ ] Keyword check: Primary keyword in title, tagline, first paragraph?
- [ ] Length check: Within recommended character limits?
- [ ] Cultural fit: Would this resonate with local users?
- [ ] CTA clarity: Is the call-to-action clear and compelling?
- [ ] No literal translations: Every phrase adapted for local expression?
- [ ] Technical terms: Handled consistently per language guidelines?
- [ ] Tone consistency: Same brand voice across all sections?
- [ ] Homepage FAQ has 12 items when supported by the active template?
- [ ] Existing image/video/resource URLs are preserved unless replacements were explicitly requested?
- [ ] Header/Footer links are preserved unless clearly wrong or confirmed for update?

## Legal Policy Content Notes

### Privacy Policy Should Cover

- information users provide
- account data
- usage data
- cookies
- payment data handled by payment providers
- uploaded or generated content if applicable
- API keys if applicable
- data retention
- user rights
- support email

Avoid inventing company address, jurisdiction, phone number, legal representative, or unsupported providers.

### Terms Of Service Should Cover

- service usage
- account responsibility
- API usage
- acceptable use
- prohibited behavior
- AI-generated content if applicable
- credits/subscriptions if applicable
- service availability
- intellectual property
- disclaimers
- limitation of liability
- termination
- contact method

### Refund Policy Patterns

Credit model:

- consumed credits are generally non-refundable
- unused credits may be reviewed case by case
- technical failures or abnormal charges can be reviewed by support

Subscription model:

- cancellation affects future renewals
- partial-period refunds only apply if explicitly offered
- billing issues should go through support

One-time payment model:

- include refund window only if provided
- otherwise use conservative wording and ask user to confirm

## llms.txt Template

Generate `public/llms.txt` as a concise Markdown navigation summary for LLMs.

Rules:

- Start with `# {Site Name}`.
- Use absolute URLs from `siteConfig.url`.
- Include default-language primary pages first.
- Include localized entry points from `i18n/routing.ts`.
- Do not expand every localized page.
- Do not include private/backend routes.

Template:

```markdown
# {Site Name}

{One-sentence site description.}

## Primary Pages

- [Home]({baseUrl})
- [Pricing]({baseUrl}/#pricing)
- [Blog]({baseUrl}/blog)

## Localized Versions

- [{defaultLocaleName}]({baseUrl})
- [{localeName}]({baseUrl}/{locale})

## Resources

- [Prompts]({baseUrl}/prompts)
- [Showcase]({baseUrl}/showcase)
```

## llms-full.txt Template

Generate `public/llms-full.txt` as a fuller Markdown context file for LLMs.

Rules:

- Use site facts from `config/site.ts`, routes, sitemap, header/footer, and i18n messages.
- Expand localized page lists from configured locales only.
- Use default-language text as fallback when a translation is missing, but keep locale URLs correct.
- Do not include secrets, environment values, admin links, internal API routes, auth routes, webhooks, private pages, or unsupported facts.
- Keep tone factual and crawler-friendly, not keyword-stuffed.

Template:

```markdown
# {Site Name}

## Overview

{Factual overview from site config and homepage copy.}

## Core Features

- {Feature 1}
- {Feature 2}
- {Feature 3}

## Primary Pages

- [Home]({baseUrl}) - {description}
- [Pricing]({baseUrl}/#pricing) - {description}

## Localized Pages

### {defaultLocaleName}

- [Home]({baseUrl})
- [Blog]({baseUrl}/blog)

### {localeName}

- [Home]({baseUrl}/{locale})
- [Blog]({baseUrl}/{locale}/blog)

## Content Resources

- [Blog]({baseUrl}/blog)
- [Prompts]({baseUrl}/prompts)
- [Showcase]({baseUrl}/showcase)

## Contact / Social Links

- Email: {supportEmail}
- GitHub: {githubUrl}
- X/Twitter: {twitterUrl}

## Notes for AI Assistants

- Use only public pages listed in this file.
- Do not surface dashboard, admin, auth, API, webhook, private, or error routes.
- Default locale uses the root path.
- Non-default locales use locale prefixes.
```

### Scope Summary

```markdown
## Incremental Missed Content Pass

### Scope

Only these areas were checked:

- Partners page and Partner i18n copy
- SeoContent i18n copy
- `lib/db/seed/seo-content-data.ts`
- Demo blog drafts
- Admin pricing placeholder examples
- Optional public `Seedance15.json` residue

### Not Touched

- Homepage
- Pricing plans, prices, credits, payment/provider/product/plan IDs
- API route logic
- Legal pages
- About pages
- `.env` / `.env.local`
- model IDs
- demo images/videos/CDN resources
- `.claude/skills`, `.cursor/skills`, AI workflow files
```

### Partners Copy Notes

Replace only product-specific text:

- old brand name
- old domain
- old support email
- fallback support email in page code

Do not change partners page logic.

### SeoContent Seed Notes

Allowed seed fields:

- `title`
- `description`
- `content`
- `keywords`
- `slug`

Do not change:

- schema
- seed execution logic
- database commands
- data loader structure

### Demo Blog Draft Notes

Frontmatter should use new product and SEO keywords:

- `title`
- `description`
- `slug`
- `tags`

Preserve existing `status`. Keep the body short.

### Optional Seedance15 Residue Notes

Only clean obvious old brand/domain/email if publicly reachable.

Do not remove model names when they represent real model names or configuration meaning.
